JSON-endepunkter per node (bytt ut http://localhost:8801)
StiBeskrivelse
/healthHeartbeat, rolle, varde-status.
/.well-known/stkPeer-discovery — node-ID, public-key, kjente peers.
/identityRegistrerte peers og revokeringer (KraftCERT-kuratert).
/peersLive helsestatus per gossip-peer.
/eventsLokal event-DB (signerte, dedupliserte).
/events/since/{ISO-tid}Events etter et gitt tidsstempel — brukes av gossip-pull.
/gossip/activity?since={t}Push/pull/discovery-aktivitet for visualisering.
/events/ingest (POST)Kollektor-grenseflate. Krever X-Ingest-Key-header.
Demo-kommandoer
# Bootstrap hele stacken (KraftCERT + Hafslund + Statkraft + kollektor + RTU)
cd demo && ./run-demo.sh
# Kjør angrepet (i en separat terminal)
RTU_URL=http://localhost:8888 ./demo/attacker/exploit.sh
# Selvtest (raskt, uten naturlige pauser)
RTU_URL=http://localhost:8888 ./demo/attacker/exploit.sh --fast
# Stopp og rydd opp
docker compose -f demo/docker-compose.demo.yml down -v
Test ingest direkte (uten RTU-sim)
curl -X POST http://localhost:8801/events/ingest \
  -H "X-Ingest-Key: demo-secret-hafslund" \
  -H "Content-Type: application/json" \
  -d '{"title":"manuell test","severity":"low","source":"syslog","scenario_id":"manual"}'
Dashboardet bruker ?seed=<url> for å peke på en alternativ KraftCERT (standard: http://localhost:8000; for demo bruk ?seed=http://localhost:8800).
